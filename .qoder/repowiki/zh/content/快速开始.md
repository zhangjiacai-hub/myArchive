## 简介

MyArchive是一个用Rust编写的多功能命令行文件压缩与解压工具。它支持将单个文件或整个目录压缩为多种格式，包括ZIP、TAR、GZIP、BZIP2、XZ、7-Zip和RAR等格式，也支持从这些格式的文件中提取内容和列出文件内容列表。该工具基于现代Rust生态系统构建，提供了简洁易用的命令行界面和强大的多格式支持能力。

## 环境要求

使用MyArchive需要满足以下环境要求：

### 系统要求
- **操作系统**: 支持Windows、macOS、Linux等主流操作系统
- **Rust版本**: Rust 1.70或更高版本
- **Cargo**: Rust包管理器（随Rust安装包一起提供）

### 运行时依赖
MyArchive在运行时依赖以下核心组件：
- **clap**: 命令行参数解析框架
- **zip**: ZIP文件处理库
- **tar**: TAR归档处理库
- **flate2**: GZIP压缩处理库
- **bzip2**: BZIP2压缩处理库
- **xz2**: XZ压缩处理库
- **sevenz-rust2**: 7-Zip格式处理库
- **unrar**: RAR格式处理库
- **walkdir**: 目录遍历库
- **anyhow**: 错误处理库

这些依赖项会在编译过程中自动下载和安装。

## 常见使用场景

### 场景一：压缩单个文件

**命令格式**:
```bash
archive compress <源文件路径> [-f <格式>] [-o <输出文件路径>]
```

**示例**:
```bash
archive compress document.pdf          # 默认ZIP格式
archive compress image.jpg -f gz       # GZIP格式
archive compress data.txt -f bz2       # BZIP2格式
archive compress log.txt -f xz         # XZ格式
```

**预期输出**:
```
已压缩到: document.pdf.zip
已压缩到: image.jpg.gz
已压缩到: data.txt.bz2
已压缩到: log.txt.xz
```

### 场景二：压缩整个目录

**命令格式**:
```bash
archive compress <目录路径> [-f <格式>] [-o <输出文件路径>]
```

**示例**:
```bash
archive compress project/              # 默认ZIP格式
archive compress docs/ -f tar          # TAR格式
archive compress logs/ -f tar.gz       # TAR+GZIP格式
archive compress backup/ -f tar.bz2    # TAR+BZIP2格式
archive compress data/ -f tar.xz       # TAR+XZ格式
archive compress files/ -f 7z          # 7-Zip格式
```

**预期输出**:
```
已压缩到: project.zip
已打包到: docs.tar
已压缩到: logs.tar.gz
已压缩到: backup.tar.bz2
已压缩到: data.tar.xz
已压缩到: files.7z
```

### 场景三：解压文件

**命令格式**:
```bash
archive extract <压缩文件路径> [-f <格式>] [-o <输出目录>]
```

**示例**:
```bash
archive extract archive.zip
archive extract backup.tar.gz -o extracted_files
archive extract data.7z -f 7z -o output
archive extract docs.rar -f rar
```

**预期输出**:
```
已解压到: archive
已解压到: extracted_files
已解压到: output
已解压到: docs
```

### 场景四：列出文件内容

**命令格式**:
```bash
archive list <压缩文件路径> [-f <格式>]
```

**示例**:
```bash
archive list documents.zip
archive list project.tar.gz
archive list backup.7z
archive list files.rar
```

**预期输出**:

## 命令行参数详解

### 主要命令

| 命令 | 描述 | 语法 |
|------|------|------|
| `compress` | 将文件或目录压缩为指定格式 | `archive compress <source> -f <format> [--output OUTPUT]` |
| `extract` | 解压指定格式的文件 | `archive extract <source> [-f FORMAT] [--output OUTPUT]` |
| `list` | 列出指定格式文件的内容 | `archive list <source> [-f FORMAT]` |

### 格式参数说明

#### 支持的格式枚举
- `zip`: ZIP压缩格式
- `tar`: TAR归档格式  
- `gz`: GZIP单文件压缩
- `tar.gz`: TAR+GZIP复合压缩
- `bz2`: BZIP2单文件压缩
- `tar.bz2`: TAR+BZIP2复合压缩
- `xz`: XZ单文件压缩
- `tar.xz`: TAR+XZ复合压缩
- `7z`: 7-Zip格式
- `rar`: RAR格式

#### 公共参数
- `-o, --output`: 指定输出文件或目录路径
- `--help`: 显示帮助信息

#### compress命令参数
- `source`: 要压缩的源文件或目录路径（必需）
- `format`: 压缩格式（可选，默认zip）
- `output`: 输出文件路径（可选）

#### extract命令参数
- `source`: 要解压的文件路径（必需）
- `format`: 压缩格式（可选，自动检测）
- `output`: 解压输出目录（可选）

#### list命令参数
- `source`: 压缩文件路径（必需）
- `format`: 压缩格式（可选，自动检测）

### 自动格式检测

MyArchive支持根据文件扩展名自动检测格式：
- `.zip`: ZIP格式
- `.tar`: TAR格式
- `.gz`: GZIP格式
- `.bz2`: BZIP2格式
- `.xz`: XZ格式
- `.tar.gz/.tgz`: TAR+GZIP格式
- `.tar.bz2/.tbz2`: TAR+BZIP2格式
- `.tar.xz/.txz`: TAR+XZ格式
- `.7z`: 7-Zip格式
- `.rar`: RAR格式

### 默认行为

1. **输出文件命名规则**:
   - ZIP: 源文件名 + `.zip`
   - TAR: 源文件名 + `.tar`
   - GZIP/BZIP2/XZ: 源文件名 + `.gz`/`.bz2`/`.xz`
   - TAR+GZIP: 源文件名 + `.tar.gz`
   - TAR+BZIP2: 源文件名 + `.tar.bz2`
   - TAR+XZ: 源文件名 + `.tar.xz`
   - 7-Zip: 源文件名 + `.7z`
   - RAR: 不支持压缩（仅解压）

2. **解压默认行为**:
   - 单文件压缩格式（GZIP/BZIP2/XZ）: 输出为去扩展名的文件名
   - 复合格式（TAR+GZIP/TAR+BZIP2/TAR+XZ）: 输出为去扩展名的目录名
   - 其他格式: 输出为去扩展名的目录名

## 故障排除

### 常见问题及解决方案

#### 1. 编译错误
**问题**: `rustc`命令找不到
**解决方案**: 
- 确保已正确安装Rust和Cargo
- 在终端中重新加载shell配置文件

**问题**: 依赖下载失败
**解决方案**:
```bash
cargo clean
cargo update
cargo build --release
```

#### 2. 格式不支持错误
**问题**: "格式不支持"或"无法自动检测格式"
**解决方案**:
```bash
# 显式指定格式
archive compress file.txt -f zip
archive extract archive.tar.gz -f tar.gz

# 检查文件扩展名
ls -la *.zip *.tar.gz *.7z *.rar
```

#### 3. 权限错误
**问题**: 无法创建输出文件
**解决方案**:
```bash
# 检查目录权限
ls -la /path/to/output/directory

# 修改目录权限
chmod 755 /path/to/output/directory

# 或者使用sudo（不推荐）
sudo archive compress file.txt -o /root/backup.zip
```

#### 4. 文件不存在错误
**问题**: "源路径不存在"
**解决方案**:
```bash
# 检查文件是否存在
ls -la /path/to/source

# 使用绝对路径
archive compress /absolute/path/to/file.txt

# 确保路径正确
archive compress ./relative/path/to/file.txt
```

#### 5. 内存不足错误
**问题**: 处理大文件时内存不足
**解决方案**:
- 分批压缩大目录
- 使用SSD存储设备
- 关闭其他占用内存的应用程序

#### 6. 权限不足错误
**问题**: 无法读取源文件或写入目标目录
**解决方案**:
```bash
# 检查文件权限
stat /path/to/source/file.txt

# 修改文件权限
chmod 644 /path/to/source/file.txt

# 检查目录权限
stat /path/to/output/directory
chmod 755 /path/to/output/directory
```

#### 7. 特定格式限制
**问题**: RAR格式不支持压缩
**解决方案**:
```bash
# RAR仅支持解压和列表
archive extract archive.rar -f rar
archive list archive.rar -f rar

# 使用其他格式替代
archive compress folder/ -f 7z
archive compress folder/ -f tar.gz
```
